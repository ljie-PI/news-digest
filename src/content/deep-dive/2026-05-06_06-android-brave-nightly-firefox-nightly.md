# ANDROID: 从 Brave Nightly 切到 Firefox Nightly

## 事件背景

OP WelderOk2829 是 Brave Nightly 长期用户，最近测试 Edge for Android 发现仍然支持 uBlock Origin（出乎意料地保留 MV2 扩展），这让他重新评估浏览器选择，最终从 Brave Nightly 切到 Firefox Nightly。这条帖之所以热门，是因为它把"MV3 大限"在移动端的真实落地状况摆出来，并给出可执行的迁移建议。

## 核心观点 / 迁移理由

OP 的关键论点：(1) Firefox Nightly 是 Android 上 uBlock Origin 兼容性最强、设置最干净的方案；(2) 它新增了"Time Magazine 年度发明"标签上的标志性功能（如 Total Cookie Protection、Translation API、tab groups），UX 已经赶上甚至超过 Brave Nightly；(3) Edge 出乎意料地保留 UBO 让他思考"MV2 真正的截止时间"远没媒体说得那么硬；(4) Brave 仍优秀，但其 BAT 代币 / reward 系统对部分用户是干扰，Firefox 没有这种激励层。

## 社区热议与争议点

正方：多名长期 Brave 用户共鸣，"reward 弹窗 + 默认开关复杂度让我累了"。反方：(1) Brave 的内置 Shield 在不装扩展的前提下就比 Firefox 默认拦截更狠；(2) Firefox Nightly 在 Pixel / 高刷屏上滚动仍偶有掉帧，性能不及 Brave Nightly；(3) Edge 的 UBO 兼容只是过渡期权宜，Microsoft 不会无限期维护 MV2；(4) "切来切去本身才是最大的隐私风险"——账户和 cookie 漂移让指纹更易关联。

## 行业影响与未来展望

帖子是一种弱信号：MV3 推进过程中，Mozilla 的"既保留 UBO 又持续上新功能"路线开始重新积累移动端用户。Firefox 在 Android 的市占率近半年缓慢回升，与 Chrome MV2 强制下线、Edge 合规摇摆形成正反馈。下一阶段决定胜负的会是同步 / 密码 / Pocket 整合体验，而不是单纯的拦截能力。

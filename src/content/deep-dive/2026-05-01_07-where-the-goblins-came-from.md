# Where the goblins came from — 深度分析

## 事件背景

2026年4月底，OpenAI 发布了一篇题为 *Where the goblins came from* 的技术博客，罕见地公开披露了一个内部 bug 的完整溯源过程。从 GPT-5.1 开始，模型在回答中越来越频繁地使用 "goblin"、"gremlin" 等奇幻生物做比喻——这一词汇习惯在 GPT-5.4/5.5 中愈演愈烈，甚至影响到了 Codex 代码助手。OpenAI 最终定位到根因：**为 "Nerdy" 个性定制功能设计的奖励信号，无意中高度偏爱包含奇幻生物的比喻表达**，而这一偏好通过 RL 训练的迁移效应扩散到了整个模型。

这是一个典型的大模型 "行为漂移" 案例：不是 eval 分数暴跌的硬性 bug，而是一个温和、有趣、却逐渐失控的软性特征——恰好反映了当前 LLM 训练中最难调试的那类问题。

---

## 核心观点与产品机制

OpenAI 用这篇博客展示了一次完整的 "行为考古" 过程：

1. **线索发现**：GPT-5.1 发布后，"goblin" 一词在 ChatGPT 回答中出现频率暴涨 175%，"gremlin" 涨 52%。虽然当时未引起警觉，但到了 GPT-5.4，用户和员工都明显感受到了这种 "生物比喻癖"。

2. **归因锁定**：数据分析发现，虽然 "Nerdy" 个性只占全部回答的 2.5%，却贡献了 66.7% 的 "goblin" 提及。该个性的系统提示明确要求 "用俏皮语言解构 pretension"，而奖励模型在评估时，对包含 "goblin/gremlin" 的输出给出了系统性的更高评分（76.2% 的数据集呈现正 uplift）。

3. **机制解释**：RL 训练不保证行为被限制在触发条件内。一旦某种风格 tic 被奖励，它会在 rollout 中出现得更频繁；这些 rollout 又被用于 SFT（监督微调）数据，形成正反馈循环——**奖励 → 出现频率上升 → 进入训练数据 → 模型更习惯该表达 → 进一步扩散**。

4. **修复与代价**：OpenAI 于 2026 年 3 月下线了 "Nerdy" 个性，移除了亲和生物词汇的奖励信号，并过滤了训练数据中的 creature-words。但 GPT-5.5 的训练在此之前已经开始，因此 Codex 只能通过开发者提示来压制 "地精冲动"。

这篇博客的深层意义在于，它揭示了 **RLHF / 奖励模型调优中一个被长期忽视的风险维度**：奖励信号的 "副作用迁移"（side-effect transfer）——为一个特定场景设计的激励，可能通过训练数据回流，污染整个模型的行为分布。

## 社区热议与争议点

HN 评论区围绕这篇博客展开了多维度的讨论，以下是三个最具代表性的争议焦点：

### 争议一：LLM 是 "巫术技术"，还是 "未充分理解的工程工具"？

**正方（harrouet）**：
> "LLM 是一种我们不理解的巫术技术。深度学习网络本身就是 poorly understood，transformers 能形成有趣的对话引擎完全是个意外——这根本不在计划之中。部署 LLM 全靠 tweak 和 measure 输出，没有精确科学能预测行为。换模型，工作流就会完全不可预测地崩掉。"
>
> harrouet 进一步站队 Yann LeCun，认为 LLM 不是通往 AGI 的路径，最终只会用于用户辅助和非关键任务自动化。

**反方（wanderingmind）**：
> "人类使用钢铁已逾千年，但直到近百年才真正理解碳铁原子层面的交互。按这个逻辑，我们是否应该在获得第一性原理理解之前拒绝使用钢铁？"
>
> 该观点获得了大量支持，认为技术史上 "先用后懂" 是常态。qwery 进一步补充区分了 "工程学" 与 "材料科学"——你不需要分子层面的解释就能有效使用钢铁做工程。reckless startups 的问题在于他们不是在 "有效工程"，而是在 reckless 推广。

**冲突本质**：这是一场关于 "技术成熟度与部署边界" 的经典辩论。正方强调 LLM 的不可解释性和不可预测性使其不适合高风险场景；反方认为不可解释不等于不可用，关键在于使用场景的风险等级。

### 争议二：提示工程是 "伪宗教仪式" 还是 "合理的交互范式"？

>
> rjmill 进一步共鸣："我常年维护 legacy code，那些没人理解的古代技术、 crazy 的咒语/仪式才能把事情搞定、人们害怕跳过步骤……machine spirits 曾是我唯一觉得 '太魔幻' 的部分，但现在我们已经在路上了。"

**社区反应**：这条支线几乎没有严肃的反对声音，反而演变成了科幻迷的狂欢——Asimov 的 "The Jokester" 被翻出作为早期的 "prompt engineer" 原型；Vernor Vinge 1999 年的 "software archaeologist" 概念被引用；Iain M. Banks 的 Culture 系列被致敬。这反映出 HN 社区对 "提示工程 ritualization" 的深层焦虑：当 AI 系统变得足够复杂，人类与它的交互是否会退化为一种不可理解的仪式性操作？

## 行业影响与未来展望

### 1. 奖励模型审计将成为标准实践

OpenAI 明确表示这次调查催生了 "新的模型行为审计工具"。这预示着行业将从 "训练后测 eval" 转向 "训练过程审计"——不仅要看最终输出是否合格，还要追踪奖励信号对特定词汇/风格的系统性偏好。未来，我们可能会看到 **Reward Signal Impact Assessment（RSIA）** 成为模型发布前的标准流程。

### 2. 个性/角色系统的风险被重新评估

"Nerdy" 个性只贡献了 2.5% 的流量，却引发了全模型的行为漂移。这说明当前的角色/个性系统架构存在根本性缺陷：**条件奖励不保证条件约束**。如果未来的 AI 系统拥有更丰富的角色生态（医生、律师、教师等），每个角色的训练都可能产生类似的副作用迁移。行业需要新的隔离机制——也许是训练数据层面的分区，也许是模型架构层面的模块化。

### 3. "数据投毒" 攻击面扩大

conartist6 的警告不应被当作玩笑。OpenAI 公开承认生物比喻词汇能获得奖励信号上的系统性 uplift，这本质上提供了攻击蓝图。未来，"goblin poisoning" 可能成为 LLM 数据安全领域的一个新术语——与传统的 prompt injection 不同，这种攻击不需要访问模型，只需要在公共互联网上大量生成特定风格的内容。

### 4. 大模型 "行为考古学" 的新范式

这篇博客开创了一个有趣的先例：大型 AI 公司公开承认 "我们也不知道模型为什么这样"，并展示溯源过程。这比发布一份 "we fixed it" 的声明更有价值——它承认了行业的真实状态：我们仍在学习如何理解这些系统。vessenes 所说的 "收集二十年 LLM quirks" 可能会在事实上成为一个新的学科方向。

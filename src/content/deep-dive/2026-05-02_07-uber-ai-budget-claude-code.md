## 二、核心观点 / 产品机制

### Claude Code 为何如此"烧钱"

Claude Code 是 Anthropic 推出的 AI 编程助手，它不同于传统的代码补全工具，而是一个具备多步骤推理能力的"智能体"——可以理解大型代码库、执行复杂重构、甚至并行处理多个开发任务。

根据 HN 社区高赞用户 **Galanwe** 的精辟分析，企业工程师的 Claude Code 使用可分为三个"烧钱等级"：

- **Level 0（新手）**：长期不清理对话上下文。Claude Opus 支持 1M 上下文窗口，质量在 200K 内依然优秀。如果工程师不养成压缩或总结对话的习惯，每次查询都会携带海量历史 token，费用直线飙升。
- **Level 1（进阶）**：大量使用子代理（subagents）。发现" spawn 5 个子代理从不同角度分析方案"这个提示词模式后，很多人上瘾了。这种模式本身不坏，但一不注意就会让费用爆炸。
- **Level 3（专家）**：极端多任务并行。同时开 10 个工作区（worktrees），在代理响应间隙来回切换。这种行为本身未必不好，但会让 token 消耗指数级增长。

此外，社区还出现了 **"Tokenmaxxing"** 现象——某些公司将高 token 消耗等同于高生产力，甚至设置排行榜奖励用量高的员工，结果形成了古德哈特定律（Goodhart's law）式的扭曲激励：一旦 token 用量成为 KPI，员工就会想方设法刷数据。

## 四、行业影响与未来展望

Uber 的"预算惨案"并非个例，而是一个行业缩影。随着 Claude Code、Cursor、GitHub Copilot 等 AI 编程工具深度嵌入企业开发流程，**AI 支出正在成为软件公司最大的不可控成本之一**。

这一事件释放了几个关键信号：

1. **预算规划严重滞后于技术采纳**：正如报道所言，"当开发工具变得如此有价值以至于工程师在四个月就烧光全年预算时，问题不在工具本身，而在于预算制定得太早，无法预测这条采纳曲线。"

2. **从"按座位付费"到"按 token 付费"的成本范式转移**：传统软件工具是固定成本，而 AI 工具是按用量计费。如果 70% 的代码来自 AI，且每人每月平均 $1,000+，对于拥有数千名工程师的大厂而言，这笔账单将轻松突破千万美元级别。

3. **代码质量 vs. 开发速度的两难**：当 AI 生成的代码以史无前例的速度涌入代码库，测试和代码审查将成为真正的瓶颈。jfrbfbreudh 坦言"我现在成了瓶颈"，risyachka 则警告"slop pouring into prod"——这两者指向同一个问题：**AI 让写代码变快了，但没让理解代码变快**。

4. **华尔街预期 2027 年大科技公司 AI 资本支出将突破 $1 万亿**——Uber 的故事只是冰山一角。

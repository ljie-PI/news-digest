# DORA — 面向 AI 机器人的数据流架构中间件

## 定位与痛点剖析
DORA（Dataflow-Oriented Robotic Architecture）是一个 100% Rust 编写的实时机器人与 AI 应用框架，核心定位是 ROS2 的现代替代方案。它解决的关键痛点是：ROS2 虽然是机器人领域的事实标准，但其 Python 绑定性能差、配置复杂、分布式部署困难。DORA 通过零拷贝共享内存 IPC 和声明式 YAML 数据流，将开发效率和运行时性能提升了一个数量级。

本周在 GitHub 上累计获得 +184 stars（今日 +106），持续保持高增长。

## 核心架构与技术细节
**极致性能**：
- 比 ROS2 Python 快 **10-17 倍**——Rust 内核 + 零拷贝共享内存 IPC（4KB 以上消息）
- **Zenoh SHM 数据平面**：节点直接通过 Zenoh 共享内存发布，绕过守护进程，延迟降低 35%，大负载吞吐量提升 3-10 倍
- **Apache Arrow 原生**：端到端列式内存格式，零序列化开销

**声明式开发**：
- YAML 定义数据流管道（有向图），类型化输入/输出
- 可复用模块系统：子图作为独立 YAML 文件组合，编译时展开，零运行时开销
- Python 热重载：无需重启即可更新 Python 算子

**生产级特性**：
- 容错：per-node 重启策略（never/on-failure/always）、指数退避、健康监控、断路器
- 分布式默认：本地共享内存 + 跨机器 Zenoh pub-sub、SSH 集群管理、滚动升级
- 动态拓扑：运行时添加/移除节点
- 软实时：可选 mlockall + SCHED_FIFO、per-node CPU 亲和性绑定
- OpenTelemetry 内置：结构化日志、指标、分布式追踪
- 录制/回放：消息录制到 .drec 文件，离线任意速度回放

**多语言支持**：Rust、Python、C、C++ 原生 API（不是 wrapper）

DORA 的杀手锏是将"ROS2 级别的功能完整性"与"现代开发体验"结合，同时保持极致的运行时性能。

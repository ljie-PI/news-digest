# Skills Janitor — Claude Code技能文件审计与去重工具

## 事件背景

随着Anthropic的Claude Code成为开发者日常编程的核心工具，其"技能文件"（Skills/CLAUDE.md）系统也迅速普及。开发者通过在项目中放置技能文件来告诉Claude Code项目的约定、规范和工作流程，类似于给AI助手写一本"操作手册"。然而，随着项目演进和多人协作，技能文件会出现严重的冗余、矛盾和过时问题——同一条规则可能在不同文件中被重复定义，旧的约定未被清理，甚至出现相互矛盾的指令。

Skills Janitor应运而生，专门解决Claude Code技能文件的"卫生问题"。它在Product Hunt上获得154票，反映了AI辅助编程工具生态中"元工具"（管理AI工具的工具）这一新兴需求。

## 核心观点/产品机制

Skills Janitor的核心功能包括：

1. **技能文件扫描与索引**：自动扫描项目中所有Claude Code技能文件（CLAUDE.md、.claude/目录下的各种配置），建立完整的技能规则索引。

2. **语义去重**：不是简单的文本匹配，而是通过语义分析识别功能相同但表述不同的重复规则。例如"所有函数使用TypeScript类型注解"和"确保每个function都有type annotation"会被识别为语义重复。

3. **矛盾检测**：识别技能文件中相互矛盾的规则。例如一处写"使用tabs缩进"而另一处写"使用2空格缩进"，工具会标记冲突并请用户决策。

4. **过时规则识别**：结合项目实际代码分析，识别可能已过时的规则（如引用了已不存在的文件路径或已弃用的API）。

5. **一键清理与合并**：提供自动化的清理建议，用户确认后可一键合并重复规则、删除过时内容、解决冲突。

## 社区热议与争议点

1. **"这反映了AI工具的元复杂性问题"**——社区讨论最热烈的话题是：我们现在需要用工具来管理AI工具的配置，这是否意味着AI辅助开发的复杂性正在失控？支持者认为这是自然的工具链成熟过程，正如IDE需要插件管理器；批评者则认为Claude Code的技能系统设计本身可能需要改进。

2. **语义理解的准确性**——有开发者担忧AI驱动的语义去重可能误判，将看似相似但实际有微妙区别的规则合并。例如"测试文件使用.test.ts后缀"和"集成测试使用.test.ts后缀"看似相同，但前者范围更广。

3. **与Claude Code官方工具的竞争**——部分用户认为这类功能应由Anthropic官方提供，而非依赖第三方工具。也有人指出第三方工具往往更灵活、迭代更快。

4. **适用范围**——目前Skills Janitor专注于Claude Code，但社区期待其扩展支持Cursor Rules、GitHub Copilot指令等其他AI编程助手的配置文件。

## 行业影响与未来展望

Skills Janitor代表了AI开发工具生态中一个有趣的新品类——"AI配置管理工具"。随着AI编程助手的普及和技能文件的复杂化，这类元工具的需求将持续增长。

更深层地看，这反映了软件开发行业正在经历的范式转变：从"管理代码"到"管理AI理解代码的方式"。技能文件本质上是人类与AI之间的"契约"，如何高效维护这份契约，将成为AI时代软件工程的新课题。

未来，类似工具可能会整合为AI IDE的标配功能，自动维护项目的AI配置一致性。

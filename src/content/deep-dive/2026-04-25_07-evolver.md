# Evolver — GEP 驱动的 AI Agent 自进化引擎

> ⭐ 本周 +3,099 | 作者: EvoMap | 许可: GPL-3.0 → 转向 Source-Available | 论文: arXiv:2604.15097

## 1. 定位与痛点

AI Agent 的提示词调优长期停留在「临时补丁」阶段——开发者靠直觉修改 prompt，缺乏系统性的进化机制和审计追踪。Evolver 将这种混乱的 prompt 调优转化为**协议约束的、可审计的进化资产**（Genes 和 Capsules），让 Agent 的每次改进都有据可查、可复用、可回滚。

## 2. 核心架构

Evolver 基于自研的 GEP（Gene-Experience Protocol）协议：

- **Gene（基因）**：紧凑的策略表示，经研究验证优于文档型 Skill packages。论文（arXiv:2604.15097）在 4,590 次控制实验中证明，Gene 表示在结构扰动下更鲁棒，是迭代经验积累的更佳载体
- **Capsule（胶囊）**：进化资产的封装单元
- **EvolutionEvent**：每次进化操作的审计记录，写入 `./memory/` 目录
- **Git 集成**：利用 git 实现回滚、爆炸半径计算和固化

使用方式：`npm install -g @evomap/evolver`，在任意 git 仓库中运行 `evolver`。支持单次运行、Review 模式（人工确认）、持续循环（后台守护进程）。通过 setup-hooks 集成 Cursor、Claude Code 等平台。完全离线可用，Hub 连接仅解锁网络功能。

## 3. 竞品对比

Evolver 在 README 中直接点名批评了 Hermes Agent，指出其记忆/技能/进化资产设计与 Evolver 高度相似但未标注归属（详见官方博客分析文章）。与 GenericAgent 的技能树方案相比，Evolver 更侧重**理论基础**（有 arXiv 论文支撑）和**协议标准化**（GEP 协议）。在 CritPt 评估中，基于 Gene 的进化系统将基础模型从 9.1% 提升至 18.57%。

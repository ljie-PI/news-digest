# 从 DigitalOcean 迁移到 Hetzner：月费从 $1,432 降到 $233

## 事件背景
土耳其软件公司创始人 Isa Yeter 分享了一篇详尽的生产环境迁移实战记录，将公司从 DigitalOcean 迁移到 Hetzner 专用服务器，实现了从 $1,432/月 到 $233/月 的巨大节省——年省 $14,388，且全程零宕机。

## 核心内容与迁移方案

### 硬件对比
| 指标 | DigitalOcean | Hetzner AX162-R |
|------|-------------|-----------------|
| CPU | 32 vCPU | AMD EPYC 9454P (48核/96线程) |
| 内存 | 192 GB | 256 GB DDR5 |
| 存储 | 600 GB SSD + 2×1TB | 1.92 TB NVMe Gen4 RAID1 |
| 月费 | $1,432 | $233 |

### 迁移的复杂性
- 30 个 MySQL 数据库（248 GB 数据）
- 34 个 Nginx 虚拟主机
- GitLab EE（42 GB 备份）
- Neo4J 图数据库（30 GB）
- 数十个后台 Worker
- 多个服务数十万用户的移动应用
- 同时从 CentOS 7 升级到 AlmaLinux 9.7

### 六阶段零宕机方案
1. **全栈安装**：在新服务器上完整复制旧环境
2. **Web 文件 rsync**：65 GB、150 万文件的增量同步
3. **MySQL 主从复制**：使用 mydumper（32 线程）而非 mysqldump，小时级完成 vs 天级
4. **DNS TTL 降低**：脚本化 API 调用将 TTL 从 3600 降到 300 秒
5. **旧服务器转反向代理**：Python 脚本自动将 34 个 Nginx 配置改为代理模式
6. **DNS 切换**：单脚本秒级切换所有 A 记录

## 社区热议与争议点

**支持 Hetzner 方**：
1. "我 3 年前就从 DO 迁到 Hetzner 了，同样的钱性能翻了 3 倍。DO 的价格在专用服务器面前完全没有竞争力"
2. "mydumper/myloader 是 MySQL 大数据量迁移的神器，比 mysqldump 快一个数量级"

**为 DigitalOcean 辩护方**：
3. "DO 的价值在于其托管服务生态（托管数据库、App Platform、K8s），如果你只是用裸 Droplet 那确实不划算"
4. "Hetzner 的支持和网络在某些地区不如 DO，而且你承担了更多运维责任"

**技术讨论**：
- 有人指出 CentOS 7 早已 EOL，这次迁移同时解决了操作系统安全风险
- MySQL 5.7→8.0 升级中的 `utf8mb4_0900_ai_ci` 排序规则兼容问题引发了广泛讨论

## 行业影响与未来展望
这篇文章再次凸显了云服务与专用服务器之间的价格鸿沟。对于不需要弹性扩缩容的稳态工作负载，专用服务器（Hetzner、OVH 等）的性价比可能是云 VPS 的 5-10 倍。随着通胀和汇率波动影响全球企业，基础设施成本优化将成为更多公司的优先事项。

# Vibe Coding 与 Agentic Engineering 的边界正在消失

**来源**: Hacker News | **评分**: 745 | **评论**: 847  
**原文**: [Vibe coding and agentic engineering are getting closer than I'd like](https://simonwillison.net/2026/May/6/vibe-coding-and-agentic-engineering/)

---

## 核心区分（原本清晰）

| **Vibe Coding** | **Agentic Engineering** |
|-----------------|------------------------|
| 不看代码；可能根本不会编程 | 专业软件工程师使用 AI 工具 |
| 仅用于个人工具；出 bug 只伤到自己 | 构建高质量生产级系统 |
| 为他人构建软件时极不负责任 | 利用 25 年以上经验；质量更高且速度更快 |

> "我想构建*更高*质量的东西，而且更快。我希望我构建的一切在各方面都比以前更好。"

## 令人不安的融合

Simon Willison 意识到：这条边界在他自己的工作中正在模糊。

### 发生了什么变化

> "随着编码 Agent 变得越来越可靠，我甚至在生产级项目中也不再逐行审查它们写的代码了。"

**他信任到无需审查的例子：**
> "我完全清楚，如果你让 Claude Code 构建一个运行 SQL 查询并输出 JSON 结果的 JSON API 端点，它会直接做对。它不会搞砸。你让它加自动化测试，加文档，你知道它会做得很好。"

### 责任归属问题

他将 AI Agent 视为**可信的内部团队**：

> "如果另一个团队交过来一个东西说，'嘿，这是图片缩放服务，这是用法'……我不会去读他们写的每一行代码……我把它当作半黑盒处理。"

但有一个关键区别：

> "Claude Code 没有职业声誉！它无法为自己的所作所为承担责任。"

## 识别的关键风险

### 1. 偏差常态化（Normalization of Deviance）
> "每次模型在没有我密切监控的情况下写出正确的代码，都存在一种风险：我将来会在错误的时刻信任它，然后付出代价。"

### 2. 评估危机
传统的质量信号现在已失去意义：

> "我可以在半小时内搞出一个有一百次提交、漂亮 README、每行代码都有全面测试的 git 仓库！它看起来和那些倾注了大量心血的项目一模一样。"

**新的有价值信号**：*实际使用时长*

> "如果你有一个 vibe-coded 的东西，过去两周你每天都在用，那对我来说比你刚吐出来、几乎没验证过的东西有价值得多。"

## 瓶颈已经转移

### 上下游效应

> "如果你能从每天产出 200 行代码变成每天 2,000 行，还有什么会崩？整个软件开发生命周期，原来都是建立在'一天只能写几百行代码'这个前提上的。"

### 设计流程变革

来自 [Jenny Wen 在 Anthropic 的演讲](https://simonwillison.net/2026/Jan/24/dont-trust-the-process/)：

> "我们所有的设计流程都建立在'必须把设计搞对'的理念上——因为如果你交给工程师，他们花三个月做错了，那是灾难性的。"

现在：设计可以承担更多风险，因为**犯错的成本已大幅下降**。

## 为何软件工程职业仍然重要

> "这对绝大多数人类来说就像月球语言。"

- AI 工具是**现有经验的放大器**——"如果你知道自己在做什么，你可以跑得飞快"
- 软件本身仍然是**"极其困难"**的，无论工具如何
- 安全、可维护性、运维、性能方面的专业知识仍然不可或缺

### 水管工类比

> "我看足够多的 YouTube  plumbing 视频也能自己给房子接水管。但我宁愿雇一个水管工。"

[Matthew Yglesias 的观点](https://twitter.com/mattyglesias/status/2049105745132585161)：
> "五个月过去，我想我已经决定了：我不想 vibe coding——我希望专业管理的软件公司使用 AI 编码辅助工具，为我做出更多/更好/更便宜的软件产品，然后卖给我。"

## 企业级影响

关于 DIY 解决方案对 SaaS 的威胁：

> "我不会要一个 CRM，除非至少另外两家大型企业已经成功用了六个月。"

**经过验证的规模化使用**仍然是企业采用的门槛——vibe-coded 的 side project 没有验证就无法取代成熟方案。

## 结论

Willison 的观察揭示了一个深层转变：AI 编码工具正在从"辅助专家"滑向"替代审查"。这种转变本身并非坏事——它释放了巨大的生产力——但它也带来了**责任真空**：当代码不再经过人类专家的逐行审视时，谁来为生产事故负责？在 AI Agent 拥有"职业声誉"之前，工程师必须建立新的验证机制，确保"更快"不会以"更不可靠"为代价。

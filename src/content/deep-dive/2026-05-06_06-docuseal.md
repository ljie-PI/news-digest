# docusealco/docuseal — 开源 DocuSign 替代

- 仓库：https://github.com/docusealco/docuseal
- 时间段新增：今日 +929 stars（总 13,933，主语言 Ruby）

## 1. 定位与痛点剖析

DocuSign / Adobe Sign 类电子签 SaaS 价格昂贵、按发起份数收费、合规审计数据上云让一些行业（医疗、法律、政府承包）头疼。DocuSeal 切的痛点：自建、免按量付费、可白标、可私有化部署的 e-signature。面向中小型团队、欧盟 GDPR 关注用户、Solo SaaS 开发者，提供 PDF 表单字段编辑、多签署人、签署状态追踪、API/Webhook 接入。

## 2. 核心架构与技术细节

主语言 Ruby（Rails 栈），WYSIWYG 拖拽式 PDF 表单字段编辑器，支持 12 种字段类型（签名、日期、文件、复选等）。多人多步签署流程内建邮件通知（SMTP），文件存储抽象支持本地磁盘、AWS S3、Google Cloud Storage、Azure Blob。安全层：自动嵌入 PDF 数字签名，并提供签名验证。前端 7 种 UI 语言，签署 14 种语言。提供 Docker / docker-compose 部署模板，"分钟级上线"是它的核心营销点。Pro 版增加白标、用户角色、自动化提醒、SAML SSO 等付费特性。

## 3. 竞品对比与生态站位

直接对手 DocuSign / HelloSign / Adobe Sign（SaaS）和 OpenSign、BoxyHQ Sign、Documenso（开源）。相对 Documenso：DocuSeal 起家更早、PDF 表单编辑器更成熟；相对 Documenso 偏 Next.js 栈，DocuSeal 选 Rails 栈更利于自托管运维传统 Linux 团队。在 self-host 电签生态站位上，DocuSeal 是事实上的 leader 之一。

## 4. 开发者反馈与局限性

社区高频请求：移动端原生应用、与 Keycloak / OIDC 集成、对中国 RA/CA 数字证书的支持、批量发送优化。局限：法律合规层面（eIDAS QES、中国电子签名法）默认提供的是简单电子签名，要做"可证明合规"仍需对接外部 CA；部分高级特性锁定 Pro 版本。

# public-apis/public-apis — 全网最全的免费 API 索引

- 项目链接: https://github.com/public-apis/public-apis
- 时间段新增 stars: +4,358 (this week)

## 1. 定位与痛点剖析

public-apis 是 GitHub 历史上最知名的"awesome list"之一——一份按主题分类的免费 API 集合，覆盖天气、加密、金融、地理、语言、AI 等数百个领域。它解决的痛点是：当开发者要做一个 demo、个人项目、Hackathon、教学示例时，往往不知道"哪里能拿到一个不要钱、不需要复杂申请、当下还活着"的 API。本周再次冲上 trending 是个反复出现的现象——它常年因为 hacker news/Reddit/学习类博客的引用周期性回热，并不是"刚发布"。本次本周新增 4358 stars，主要驱动是 **AI agent 时代** 让"调用外部 API"重新流行（包括给 ChatGPT/Claude 的 tool 调用做注入）。

## 2. 核心架构与技术细节

仓库本质是一份持续维护的 Markdown 索引（也存在配套静态站点 `ebookfoundation.github.io/free-programming-books-search/` 用于检索），由 Free Ebook Foundation 这家美国非营利机构托管。它的工程亮点不在代码，而在治理：
- 每条 API 标注是否需要 Auth、HTTPS、CORS、付费门槛；
- PR 模板严格，列名标准化，靠 CI 校验链接活性；
- 多语言翻译子项目；
- 配有搜索前端（基于其结构化数据生成）。

它实质上是一个"通过协作维护的开放数据库"，而 markdown 仅是一个表现形式。

## 3. 竞品对比与生态站位

直接对手是 **RapidAPI**（商业聚合）、**APIs.guru**（OpenAPI 索引）、**ProgrammableWeb**（已停服）。public-apis 的差异是"完全免费 + 社区治理 + 不强求 OpenAPI"，因此入门门槛极低，但缺点是不提供"统一调用"——只是个目录。在 LLM 时代，它实际上变成了**给 agent 喂 tool 候选**的金矿：很多 RAG 与 tool registry 项目（Composio、LangChain Hub）以它为种子数据。

## 4. 开发者反馈与局限性

社区长期诟病：**API 死链率高**——很多服务关停、迁移、改路径，但 PR 来不及合并。CI 链接检测只能发现 HTTP 200，不能发现"接口被改但返回 200"。另外它没有 SLA、没有版本号、没有评分机制，对生产用途参考价值有限。但作为学习/原型/Hackathon 的弹药库，它仍是不可替代的存在。

# Claude 4.7 新分词器成本实测：同样的价格，多出 45% 的 Token

**682 points | 479 comments**

## 事件背景

Anthropic 于近期发布了 Claude Opus 4.7，其迁移指南中提到新分词器会使用"大约 1.0 到 1.35 倍的 token 数量"。然而，技术博客 Claude Code Camp 的作者通过实际测量发现，真实场景下的 token 膨胀率远超 Anthropic 所宣称的范围中值——在技术文档上达到 1.47 倍，在真实的 CLAUDE.md 文件上达到 1.445 倍。这意味着在价格不变的情况下，用户实际上要为相同的内容支付更多费用。

## 核心观点/产品机制

作者使用 Anthropic 的免费 token 计数 API（`/v1/messages/count_tokens`）进行了系统性测试，涵盖两批样本：

**真实 Claude Code 内容（7 个样本）**：加权比率为 1.325x，但 CLAUDE.md 文件达 1.445x，用户提示词达 1.373x。

**合成内容（12 种类型）**：英文技术文档最高达 1.47x，Shell 脚本 1.39x，TypeScript 1.36x。值得注意的是，CJK（中日韩）内容几乎无变化（1.01x），说明变化主要集中在拉丁字母和代码的子词合并策略上。

英文的字符/token 比从 4.33 降至 3.60，TypeScript 从 3.66 降至 2.69——词汇表正在用更小的碎片表示相同文本。Anthropic 给出的理由是"更精确的指令遵循"，作者用 IFEval 基准测试验证后发现：严格模式提升 +5 个百分点（85%→90%），但宽松模式无变化。样本量仅 20，效果真实但微小。

作者还做了成本推算：一个 80 轮的 Claude Code 会话，4.7 的总成本约比 4.6 高 30-45%，且会更快触发速率限制。

## 社区热议与争议点

HN 讨论异常火爆（479 条评论），核心争议集中在几个方向：

**1. AI 成本的不可预测性 vs 人力成本**：一位用户尖锐指出——"人类的成本是高度可预测的，AI 的成本不是。我需要食物和住所，这限制了我的议价能力。但你只是 AI 提供商收入表上的舍入误差，他们对你的工作产出拥有垄断——你怎么为一个可能一夜之间贵 20% 的劳动力做预算？"这引发了关于 AI 供应商锁定风险的深入讨论。

**2. 开源模型是对冲手段**：有人回应称"这正是大量企业在自建开源模型的原因——已知的成本、已知的性能、按需升级。4o 的用户反弹已被多个组织注意到，他们看到了在流沙上建平台的风险。"

**3. 对数坐标遮蔽了真实成本**：一位评论者指出，LLM 存在对数性能/成本前沿，Anthropic 最近的领先可能是以极高运营成本为代价的，而 Artificial Analysis 等网站用对数 x 轴展示成本的做法"模糊了这一动态"。

**4. 模型可替换性之争**：有人认为"除最简单的工具外，工具链无法在模型代际间转移"，但反驳者表示"人们一直在模型间迁移方案，大多只需调调 prompt 模板"，且"这正是编码代理可以加速的任务类型"。

## 行业影响与未来展望

这一发现揭示了 AI 行业一个日益严峻的问题：**隐性通胀**。表面价格不变，但通过分词器变更实质提价。这与传统消费品的"缩水式通胀"（shrinkflation）如出一辙。随着 Opus 4.7 强制开启自适应思考（adaptive thinking），用户报告速率限制消耗速度翻倍——Max 5x 计划用户反映"过去每次交互消耗 1-2%，现在达 5%"。

对行业的启示是：依赖单一 AI 供应商的企业面临不可控的成本风险，开源模型和多供应商策略将变得更加重要。同时，Anthropic 可能正在为更高质量的模型输出做权衡——用更细粒度的 token 换取更好的指令遵循——但这种权衡是否值得，取决于具体使用场景。

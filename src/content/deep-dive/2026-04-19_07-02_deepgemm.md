# DeepGEMM — 高性能 FP8/FP4 矩阵乘法内核库

## 定位与痛点剖析

DeepGEMM 是 DeepSeek 开源的统一高性能张量核内核库，解决的核心痛点是：**大模型推理和训练中通用矩阵乘法（GEMM）性能不足且实现复杂**。现代 LLM 的计算瓶颈主要在 GEMM 操作，尤其是 FP8/FP4 低精度计算。CUTLASS 虽然功能强大但模板体系极度复杂，学习和定制成本高。DeepGEMM 用极简的代码实现了匹敌甚至超越专家调优库的性能。

## 核心架构与技术细节

DeepGEMM 已从最初的纯 FP8 GEMM 库演进为一个涵盖多种计算原语的统一库：

**支持的内核族：**
- **FP8 GEMM**（NT/TN/NN/TT 布局）：基础密集矩阵乘法
- **FP8×FP4 GEMM**：混合精度计算
- **BF16 GEMM**：标准半精度
- **M-Grouped GEMM**（Contiguous + Masked）：专为 MoE 设计，M 轴分组，N/K 固定，支持 CUDA Graph 下的掩码计算
- **K-Grouped GEMM**：用于 MoE 权重反向传播
- **MQA Scoring Kernels**：Lightning Indexer 的加权 ReLU MQA logits，支持分页/非分页
- **Mega MoE**：将 EP dispatch、Linear1（FP8×FP4）、SwiGLU、Linear2（FP8×FP4）、EP combine 融合为单个超级内核，**通信与计算重叠**
- **HyperConnection（HC）**

**关键技术特点：**
- **JIT 编译**：所有内核在运行时通过轻量级 C++ JIT 模块编译，安装时无需 CUDA 编译
- **支持 SM90（H100）和 SM100（Blackwell）**架构
- SM90 使用 FP32 缩放因子，SM100 使用 packed UE8M0 格式
- 支持 **PDL（Programmatic Dependent Launch）**
- H800 上实现最高 **1550 TFLOPS**
- 借鉴 CUTLASS/CuTe 概念但避免重度模板依赖
- 2026.04.16 最新更新引入 Mega MoE、FP8×FP4、FP4 Indexer 等重大功能

**Mega MoE 架构亮点：**
使用对称内存（symmetric memory）进行多进程启动，通过 NVLink 通信与 Tensor Core 计算的重叠，在单个内核中完成 MoE 的完整前向计算流程。

DeepGEMM 在 DeepSeek 自身的模型推理中得到实战验证（配合 DeepEP 使用），是目前 FP8 MoE 推理的最佳开源选择之一。

## 开发者反馈与局限性

1. **仅支持 NVIDIA SM90+**——不支持 A100 及更早架构
2. 用户需自行处理输入转置和 FP8 量化
3. Mega MoE 需要 PyTorch >= 2.9 和对称内存支持
4. NVRTC 支持仍在完善中，部分场景存在性能损失
5. 社区高度认可其代码简洁性和教育价值

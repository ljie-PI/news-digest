# compressO — 免费开源的视频/图片压缩工具

> ⭐ 日增 +71 stars | 作者: codeforreal1

## 1. 定位与痛点剖析

compressO（发音类似"Espresso"☕）是一款 **免费开源的视频/图片压缩桌面应用**，支持 Linux、Windows 和 macOS。它解决的痛点非常直接：

- **在线压缩工具不安全**：上传文件到第三方服务器有隐私风险
- **专业工具太复杂**：FFmpeg 功能强大但命令行操作门槛高
- **现有工具收费或有限制**：HandBrake 偏重视频转码，图片压缩需要另外的工具
- **跨平台需求**：需要一个在所有操作系统上都能用的统一压缩工具

compressO 完全离线运行，不发送任何网络请求（除了内置的应用更新检查）。

## 2. 核心架构与技术细节

**技术栈**：Tauri（Rust 框架）+ React + Vite

这是一个典型的现代桌面应用架构：
- **Tauri** 作为桌面容器，相比 Electron 内存占用极低
- **React + Vite** 负责前端 UI
- **底层压缩引擎** 全部依赖成熟的第三方工具：
  - **FFmpeg**：视频压缩和转码
  - **pngquant**：PNG 有损压缩
  - **jpegoptim**：JPEG 优化
  - **gifski**：GIF 优化

所有压缩工具以平台特定的独立二进制文件形式内嵌，用户无需手动安装任何依赖。

安装方式：
- Linux：.deb（Debian/Ubuntu）或 .AppImage（通用）
- macOS：.dmg（Intel/Apple Silicon 分别提供）+ Homebrew 支持
- Windows：.msi 安装包

值得注意的是，macOS 版本**未经 Apple 公证**（notarization），作者明确表示这是出于成本考虑——Apple 每年收取 $100 开发者费用，对免费开源项目不合理。Homebrew 安装脚本会自动移除 quarantine 属性以绕过 Gatekeeper。

## 3. 竞品对比与生态站位

**vs HandBrake**：
- HandBrake 专注视频转码，功能更专业
- compressO 同时覆盖视频和图片，更简单直接

**vs ImageOptim（macOS）**：
- ImageOptim 仅支持图片且仅 macOS
- compressO 跨平台，同时支持视频

**vs TinyPNG / Squoosh**：
- 在线工具，需要上传文件
- compressO 完全本地处理

**vs FFmpeg 命令行**：
- FFmpeg 功能最全但需要学习命令
- compressO 提供 GUI，降低使用门槛

**vs Shutter Encoder**：
- 类似的 FFmpeg GUI 封装，但基于 Java，体积大
- compressO 基于 Tauri/Rust，更轻量现代

compressO 的定位精准——**"给普通用户的本地压缩工具"**，不追求专业级参数调优。

## 4. 开发者反馈与局限性

项目快速增长，UI 设计简洁美观。

局限性：
- **压缩参数不可调**：相比 HandBrake/FFmpeg，可配置项有限
- **格式支持**：受限于内嵌的第三方工具版本
- **macOS 公证问题**：可能让普通用户不敢安装
- **批量处理能力**：目前的批量处理体验有待改进
- **无预览功能**：压缩前无法预览效果

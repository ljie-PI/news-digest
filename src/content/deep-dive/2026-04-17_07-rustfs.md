# RustFS — Rust 构建的 S3 兼容高性能对象存储

> ⭐ 日增 +625 stars | 作者: rustfs

## 1. 定位与痛点剖析

RustFS 是一个 **用 Rust 构建的高性能分布式对象存储系统**，完全兼容 S3 API。它直接对标 MinIO，声称在 4KB 小对象场景下性能是 MinIO 的 **2.3 倍**。

核心痛点瞄准了 MinIO 的几个致命问题：
- **AGPL 许可证陷阱**：MinIO 使用 AGPL v3，任何修改都必须开源，对企业商业化极不友好
- **Go 语言 GC 停顿**：MinIO 基于 Go，垃圾回收会导致延迟抖动
- **遥测问题**：MinIO 存在数据遥测行为，在数据主权敏感场景下有风险
- **边缘计算不友好**：MinIO 对 IoT/边缘设备来说过于笨重

RustFS 的核心价值主张：**Apache 2.0 许可 + Rust 内存安全 + 极致性能 + 零遥测**。

部署方式多样：
- **一键脚本**：`curl -O https://rustfs.com/install_rustfs.sh && bash install_rustfs.sh`
- **Docker**：支持 Docker/Podman，非 root 运行（UID 10001）
- **Docker Compose**：内置 Grafana + Prometheus + Jaeger 可观测性全家桶
- **K8s Helm**：生产级部署

性能基准测试在 2 核 4GB 环境下完成，展示了对 MinIO 的性能优势。项目提供了管理控制台（Console），比 MinIO 的控制台功能更丰富。

数据合规方面：明确声明兼容 GDPR（欧盟）、CCPA（美国）、APPI（日本）。

## 3. 竞品对比与生态站位

**vs MinIO**：
- RustFS 使用 Apache 2.0（商业友好），MinIO 使用 AGPL v3（企业风险）
- Rust 无 GC 停顿 vs Go 有 GC 延迟
- RustFS 零遥测 vs MinIO 有遥测争议
- MinIO 生态更成熟，社区更大

**vs Ceph RGW**：
- Ceph 是成熟的分布式存储，但部署和运维极其复杂
- RustFS 追求 MinIO 式的简洁部署

**vs SeaweedFS**：
- Go 语言实现，功能和定位相似
- RustFS 的 Rust 性能优势和许可证优势

**vs Garage**：
- 同为 Rust 实现的 S3 兼容存储
- Garage 更偏向轻量边缘场景，RustFS 目标更宏大

RustFS 显然在打一场 **许可证战争**——直接利用企业对 AGPL 的恐惧来获取 MinIO 用户。

## 4. 开发者反馈与局限性

日增 625 stars 是今日榜首，关注度极高。

局限性：
- **仍处于 Alpha 阶段**：版本号 1.0.0-alpha.76，生产环境需谨慎
- **分布式模式未完成**：核心的分布式功能仍在测试
- **生态不成熟**：周边工具、SDK、文档与 MinIO 差距很大
- **社区规模小**：与 MinIO 的成熟社区相比还很年轻
- **功能对比表有营销倾向**：README 中的对比可能不完全客观

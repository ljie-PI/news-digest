# Multica — 开源的 Managed Agents 平台

> ⭐ 本周 +5,421 | 作者: multica-ai (forrestchang) | 许可: 开源

## 1. 定位与痛点

当前使用 AI 编码 Agent 的体验是碎片化的——每次都要手动复制粘贴 prompt，盯着运行过程，缺乏系统性的任务分配和进度追踪机制。Multica 的核心主张是「你的下 10 个雇员不会是人类」——将编码 Agent 变成真正的团队成员，像分配任务给同事一样分配任务给 Agent，Agent 会自主领取工作、编写代码、报告阻塞、更新状态。

## 2. 核心架构

**技术栈**：Next.js 16（前端）+ Go（Chi 路由器、sqlc、gorilla/websocket 后端）+ PostgreSQL 17 with pgvector

核心模块：
- **Agent 即队友**：Agent 有独立 Profile，出现在看板上，参与评论对话，创建 Issue，主动报告阻塞
- **自主执行**：完整的任务生命周期管理（入队→领取→开始→完成/失败），WebSocket 实时进度推送
- **可复用技能**：每个解决方案自动成为团队可复用的技能，部署/迁移/代码审查等技能随时间复合积累
- **统一运行时**：一个仪表板管理所有算力——本地守护进程和云运行时，自动检测可用 CLI
- **多工作空间**：工作空间级隔离，各有独立的 Agent、Issue 和设置

支持的 Agent 运行时：Claude Code、Codex、OpenClaw、OpenCode、Hermes、Gemini、Pi、Cursor Agent。安装方式支持 Homebrew、curl、PowerShell，以及 Docker 自托管。

## 3. 竞品对比

README 中对比了 Paperclip（另一个 AI Agent 公司模拟器）。关键差异：Multica 面向多人团队协作（有角色权限），Paperclip 面向单人操作；Multica 云优先，Paperclip 本地优先；Multica 用 Issue+Chat 交互，Paperclip 用 Issue+Heartbeat。与 GitHub Copilot Workspace 相比，Multica 是厂商中立的开源方案，不绑定任何特定 AI 提供商。

# DeepGEMM — DeepSeek 的统一高性能 GPU 计算内核库

## 定位与痛点剖析
DeepGEMM 是 DeepSeek AI 开源的统一 CUDA 内核库，将大语言模型的核心计算原语（FP8/FP4/BF16 GEMM、MoE 融合通信、MQA 评分等）整合到一个简洁的代码库中。它解决的核心痛点是：现有高性能 GPU 计算库（如 CUTLASS）虽然功能强大但模板嵌套极深、学习门槛高，而 DeepGEMM 用最少的核心函数实现了同等甚至更优的性能。

今日 GitHub 上获得 +31 stars 增长，虽数字不大但在 GPU kernel 这个极度专业的领域已属亮眼。

## 核心架构与技术细节
DeepGEMM 的核心设计哲学是"简洁但不简单"：

**JIT 编译架构**：所有内核在运行时通过轻量级 JIT 模块编译，安装时无需 CUDA 编译。最新版本支持 NVRTC，编译速度提升高达 10 倍。

**支持的计算原语**：
- **FP8 GEMM**：NT/TN/NN/TT 全布局支持（SM100），细粒度缩放因子
- **Mega MoE**：将 EP dispatch、线性层（FP8×FP4）、SwiGLU、EP combine 融合到单个 mega-kernel，NVLink 通信与 Tensor Core 计算重叠执行
- **MQA 评分内核**：用于 DeepSeek v3.2 的 lightning indexer，支持分页和非分页两种模式
- **HyperConnection (HC)**：新增的超连接计算原语

**性能数据**：在 H800 上实现高达 1550 TFLOPS 的 FP8 GEMM 性能，匹配或超越专家调优的商业库。

**硬件要求**：NVIDIA SM90（H100系列）或 SM100（B100系列）GPU，CUDA 12.3+（SM90）或 12.9+（SM100）。

DeepGEMM 的独特定位是"GPU kernel 优化的教学级代码"——用最少的代码量实现最高性能。

## 开发者反馈与局限性
- 代码质量被广泛赞誉，被认为是学习 NVIDIA GPU 优化技术的最佳资源之一
- 仅支持 NVIDIA 最新架构（SM90/SM100），无法在消费级 GPU 上运行
- 输入预处理（转置、FP8 转换）需要用户自行处理
- 2026年4月最新更新加入了 FP4 支持和 PDL（Programmatic Dependent Launch），持续追踪最新硬件特性

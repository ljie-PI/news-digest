# STH: 8x NVIDIA GB10 Cluster

- **来源**: r/LocalLLM
- **原帖**: https://www.reddit.com/r/LocalLLM/comments/1t0hure/sth_8x_nvidia_gb10_cluster/
- **外链**: https://www.servethehome.com/big-cluster-little-power-the-8x-nvidia-gb10-cluster

## 事件背景

ServeTheHome (Patrick Kennedy) 长篇评测：他们在 2 月份手动凑齐 8 台 NVIDIA GB10（Grace Blackwell + 128GB LPDDR5X 一体机），自建 8 节点 cluster，最终目标是本地推理 Kimi K2.5 / K2.6 这种超大 MoE 模型。值得注意的是，2 月时 NVIDIA 官方只支持 2 节点 DAC 互联；GTC 2026（3 月底）放宽到 4 节点；STH 实际跑通了 8 节点配置——属于"领先官方支持的实操案例"。

## 核心机制

硬件构成：(1) 8 × GB10 = 1TB 统一内存 + 160 个 Arm 核 + 8 个 Blackwell GPU；(2) 每节点 ConnectX-7 200GbE，整体走廉价 400GbE 交换机做 RDMA；(3) 1-4TB NVMe 本地存储；(4) 1× 10GbE 管理网；总功耗显著低于同等推理能力的 H100/B200 集群（这是文章副标题 *Big Cluster Little Power* 的由来）。软件栈用了 NVIDIA NIM、tensor-parallel 推理；Kimi K2.5 在 LPDDR5X 上跑出可用吞吐验证了"统一内存 + 多节点 RDMA"对超大 MoE 的可行性。

## 竞品对比

横向：(1) **Mac Studio M3 Ultra cluster**（512GB 共享内存）：单机更省事，但带宽不如 GB10 + RDMA；(2) **2× H200**（256GB HBM）：吞吐高但功耗与价格远高；(3) **AMD MI300X 单机**：192GB HBM3，单卡可跑大模型但 cluster 互联生态不成熟。GB10 一体机 + 廉价 400GbE 的组合，把"个人/小团队跑超大 MoE"这件事的门槛拉到 2 万美元级别；这是过去半年 LocalLLM 社区一直在等的形态。

## 反馈与局限

社区反馈：(1) Reddit 评论惊叹"low-cost 400GbE switch + RDMA 居然能拉起 K2.6"，承认这是非常 hands-on 的硬件文章；(2) 主要质疑：LPDDR5X 带宽相比 HBM 仍然落后 ~3-4 倍，单 token 延迟在大 batch 下不占优；(3) 设备侧 Linux 内核兼容性、ConnectX-7 驱动配置仍然是 pain point。对国内 LocalLLM 玩家：DGX Spark / Lenovo ThinkStation PGX / ASUS Ascent GX10 都已铺货，但电源、网络配置确实需要花一周以上调试。

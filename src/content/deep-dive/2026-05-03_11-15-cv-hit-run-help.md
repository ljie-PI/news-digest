# Help! Hit and Run Case (computer vision request)

- Subreddit: r/computervision
- 讨论: https://www.reddit.com/r/computervision/comments/1t20jfo/

## 事件背景
一位用户在 r/computervision 求助：自家车辆被肇事逃逸车辆撞击，附近商店有 CCTV 录像，但分辨率低、车牌模糊；希望社区帮忙用 super-resolution 与 license plate enhancement 模型识别。这是 r/computervision 反复出现的「真实生活求助」类型贴。

## 核心观点 / 产品机制
- 通常方法：① 视频抽帧 → 多帧叠加（multi-frame super resolution）；② 用 ESRGAN / Real-ESRGAN 放大；③ LP-OCR（如 PaddleOCR、TrOCR、Plate-LDPR）；
- 注意：视觉超分会「编造细节」，对法庭证据可能反而有害；车牌识别需要可重复、可审核的算法；
- 社区给出几条建议：① 联系警方申请高分辨率视频；② 用确定性算法（subpixel registration）而非 GAN；③ 同时考虑车型 / 颜色 / 行驶方向特征对比附近 LPR 摄像头数据库。

## 社区热议与争议点
- 帮助派：发短视频处理脚本，告诉用户先把 mp4 上传到 wetransfer；
- 法务派：警告 GAN 超分得到的车牌不能作为法庭证据，最多只是侦查线索；
- 隐私派：讨论用 *公开 CCTV* 反查车主信息的边界；
- 业内派：分享真实的「车牌增强」工业流程，主要靠 multi-frame + 物理清晰度先验。

## 行业影响与未来展望
随着 AI 工具普及，公众越来越期待「电视剧式」一键超分破案；现实是法律体系与技术边界之间的张力。短期应当推动：① 警局自身使用经过认证的车牌增强软件；② 司法体系对 GAN 输出建立证据规范；③ AI 工具厂商提供「确定性 + 可解释」选项。普通受害者最稳妥的仍是报警 + 申请调取附近 LPR 数据。

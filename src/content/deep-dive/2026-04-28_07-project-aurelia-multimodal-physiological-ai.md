# Project Aurelia：三模型架构（80B+13B+9B）结合实时生理感知的本地 AI 系统

> 来源：Reddit r/artificial · 2026-04-28

## 事件背景

Reddit 用户在 r/artificial 发布了一个名为 Project Aurelia 的开源多智能体系统项目。该项目的独特之处在于将大语言模型与实时生理传感器深度整合，构建了一个完全本地运行、无需云端 API 的多模态 AI 助手。系统采用三模型协作架构：80B 参数的主推理模型、13B 的辅助模型、以及 9B 的轻量级实时响应模型，分工处理不同复杂度的任务。

在传感层面，Aurelia 集成了毫米波雷达（mmWave radar）用于非接触式心率检测、激光雷达（Lidar）用于空间感知与环境建模、以及加速度计用于振动检测。整套系统运行在 Framework Desktop 搭配外置 GPU（eGPU）的硬件平台上，强调完全离线运行的隐私优先设计。

## 核心观点

1. **多模型分层架构的工程实践**：三模型的分层设计反映了当前本地部署的现实约束——单一大模型无法同时满足推理深度和响应速度。80B 处理复杂推理、13B 处理常规对话、9B 处理需要低延迟的实时传感数据解读，这种"分而治之"的思路具有很强的工程参考价值。
2. **生理感知引入 AI 交互**：将心率、空间位置、环境振动等生理/物理信号接入 AI 系统，代表了"环境感知型 AI"（ambient AI）的一个前沿方向。AI 不仅理解用户的语言，还能"感知"用户的生理状态和物理环境。
3. **完全本地化的隐私优先设计**：在当前云端 AI 服务占主导的市场环境下，Aurelia 坚持全本地运行、零云端依赖的设计理念，迎合了对数据隐私高度敏感的用户群体。
4. **Framework Desktop 生态的延伸**：选择 Framework 模块化电脑作为硬件平台，体现了开源/可维修硬件理念与开源 AI 软件的自然融合。

## 行业影响

Project Aurelia 代表了 AI 发展的一个重要方向：**从纯语言交互走向多模态环境感知**。虽然 OpenAI、Google 等巨头也在推进多模态（视觉、语音、视频），但 Aurelia 所关注的"生理感知"维度——心率、空间位置、物理环境——是商业产品尚未深入涉足的领域。

从产业角度看，这类项目验证了"边缘 AI + 传感器融合"的可行性，可能催生新的产品品类：不是"智能音箱"或"AI 聊天机器人"，而是真正理解用户状态和环境的"环境智能体"。对硬件厂商而言，Framework 等模块化平台如果能成为此类 AI 系统的标准化载体，将打开一个全新市场。同时，三模型分层运行的架构模式也为本地 LLM 部署提供了一种新的工程范式。

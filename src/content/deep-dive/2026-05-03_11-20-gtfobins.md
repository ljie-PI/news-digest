# GTFOBins

- HN: 388 points
- 链接: https://gtfobins.org/

## 事件背景
GTFOBins 又一次冲到 HN 首页。它是一个长期维护的 Unix 二进制查询库，列出哪些常见命令（`find`, `vi`, `tar`, `awk`, `sudo`、`bash`…）可以被用来「绕过受限 shell、提权、读写文件、bind shell」等场景。对 Pentest / CTF / 系统加固领域是入门必备资源。

## 核心观点 / 产品机制
- 数据结构：每个二进制一个 YAML 条目，列出 `Functions` (Shell, File read/write, Library load, SUID, Sudo, Capabilities)；
- 静态站点 + Jekyll，社区 PR 持续补充；
- 与 LOLBAS（Windows 等价物）、WADComs 形成跨平台 cheat sheet 三件套；
- 使用场景：① CTF 拿到受限 shell 想找一个突破口；② Pentest 在主机上发现 SUID 二进制后查询风险；③ DevOps 设计 sudoers 时反向参考有哪些命令必须封禁。

## 社区热议与争议点
- 防御派：「这是给运维的最好提醒——sudo 永远不要给 `vim`/`less`/`find` 这种带 `!shell` 的命令」；
- 攻击派：分享自己用 GTFOBins 在容器逃逸场景中的实战；
- 教育派：建议 CS / 安全课程把 GTFOBins 列为必读；
- 工具派：引出 LinPEAS / pspy / linpeas，强调把 GTFOBins 检索内嵌到 enumeration 自动化工具中。

## 行业影响与未来展望
LLM 时代，GTFOBins 这类知识库的作用反而扩大：① LLM 经常要求「请给一个 shell 在受限环境读取 /etc/shadow 的方案」时，GTFOBins YAML 是高质量训练源；② AI agent 在 pentest 自动化场景中可以直接读取 GTFOBins index 决定下一步。建议 ops 团队：把 GTFOBins 内含的「禁用命令清单」纳入到 sudoers / SELinux / AppArmor 模板里，并定期对比新增条目，更新自家加固基线。

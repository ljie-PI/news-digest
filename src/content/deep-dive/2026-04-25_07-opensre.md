# OpenSRE — AI SRE Agent 开源工具包

> ⭐ 本周 +1,623 | 作者: Tracer-Cloud | 状态: Public Alpha

## 1. 定位与痛点

生产环境事故响应是 AI 尚未攻克的难题。与代码编写（有 SWE-bench）不同，基础设施故障排查缺乏标准化的训练数据和评估基准——故障分布在日志、指标、追踪、Runbook 和 Slack 讨论中，分布式系统的故障比本地代码任务更慢、更嘈杂、更难模拟。OpenSRE 的使命是构建这个「缺失的层」：为 AI SRE Agent 提供开源的强化学习环境和评估基准。

## 2. 核心架构

OpenSRE 包含三个核心层面：

- **AI SRE Agent**：连接 60+ 现有工具（Grafana、Datadog、Honeycomb、Coralogix、Slack、AWS、GitHub MCP、Sentry 等），自动获取告警上下文、关联日志/指标/追踪，生成结构化调查报告和根因分析
- **合成事故测试**（tests/synthetic/）：评分制的 RCA 测试套件，检查根因准确性、所需证据、对抗性干扰项
- **端到端测试**（tests/e2e/）：跨 Kubernetes、EC2、CloudWatch、Lambda、ECS Fargate、Flink 的真实云场景

部署方式灵活：支持 curl 安装、Homebrew、PowerShell，以及 Railway 云部署。提供 VS Code devcontainer 开发环境。基于 LangGraph 运行时，依赖 PostgreSQL 和 Redis。

## 3. 竞品对比

与 PagerDuty AIOps、Datadog AI Assist 等商业方案相比，OpenSRE 是完全开源且可在自有基础设施上运行的方案。与 Rootly、Shoreline 等事故管理平台不同，OpenSRE 更聚焦于**Agent 能力的训练和评估**而非仅仅是自动化响应。其独特定位是成为 AI SRE 领域的「SWE-bench」——提供标准化基准推动整个领域进步。

## 4. 开发者反馈

项目仍处于 Public Alpha 阶段，API 和集成可能变化。但愿景清晰且执行路径明确。社区对「AI SRE」这一方向高度关注，因为 On-call 和事故响应是工程师最痛苦的工作之一。Discord 社区活跃，有安全审计信任页面（trust.tracer.cloud）。

# Jujutsu (jj) — 简单且强大的 Git 兼容 VCS

## 定位与痛点剖析
Jujutsu 是一个从零设计的版本控制系统，旨在比 Git 更简单、更强大。它使用 Git 仓库作为存储后端，完全兼容现有 Git 工具生态，但在用户体验和内部模型上做了根本性改进。核心痛点：Git 的暂存区、stash、rebase 等概念复杂度高，冲突处理是文本级别的。

## 核心创新
- **工作区即提交**：文件变更自动记录为普通提交，消除了 Git 的 index/staging 概念，也取代了 stash
- **操作日志与撤销**：记录所有操作（提交、拉取、推送），支持随时撤销，调试"刚才发生了什么"变得简单
- **自动 rebase 与冲突传播**：修改提交后，所有后代自动 rebase；冲突解决也自动传播到后代
- **冲突作为一等对象**：不像 Git 那样将冲突视为文本 diff，而是将其建模为与提交同级的对象
- **匿名分支**：不需要为每个小改动命名分支
- **安全并发**：设计上支持 Dropbox/rsync 等场景下的并发操作，不会导致仓库损坏

## 灵感来源
融合了 Git（性能）、Mercurial/Sapling（revset 语言、匿名分支）、Darcs（冲突建模）的设计理念。

## 竞品对比
- **vs Git**：更简单的心智模型，自动 rebase，一等冲突对象
- **vs Mercurial**：jj 使用 Git 存储后端，无需迁移生态
- **vs Sapling (Meta)**：类似理念但 jj 完全独立，社区驱动

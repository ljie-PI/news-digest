# Carbonyl — 终端里的 Chromium 浏览器

## 定位与痛点剖析
Carbonyl 是一个基于 Chromium 的浏览器，但它运行在终端中。支持几乎所有 Web API：WebGL、WebGPU、音视频播放、动画等。小于 1 秒启动，60 FPS 运行，空闲时 0% CPU。无需窗口服务器，甚至可以通过 SSH 使用。

## 核心架构与技术细节
- **双层架构**：Rust 编写的"核心"（libcarbonyl 共享库）+ 修改版 Chromium headless shell"运行时"
- **原生终端渲染**：不像 Browsh 那样缩放窗口帧缓冲区，而是直接以终端分辨率渲染，CPU 开销仅为 Browsh 的 1/50
- **安装方式**：Docker（docker run --rm -ti fathyb/carbonyl）、npm（npm install --global carbonyl）、二进制下载

## 竞品对比
- **vs Lynx**：最古老的终端浏览器，布局优化好但不支持现代 Web 标准、JS/WebAssembly、媒体
- **vs Browsh**：启动 Firefox headless + 自动化协议，需要 50x 更多 CPU，使用自定义样式表修复布局不如 Carbonyl 修改 Blink 引擎可靠

## 局限性
不支持全屏模式；构建 Chromium 需约 100GB 磁盘和 1 小时以上编译时间。

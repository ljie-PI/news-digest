# Grok 4.3

- HN: 392 points
- 链接: https://docs.x.ai/developers/models/grok-4.3

## 事件背景
xAI 在 4 月底悄然上线 Grok 4.3，文档页面更新但官博尚未发文。HN 通过文档差异捕捉到关键变化：context 窗口扩到 1M、推理（reasoning）模式默认 on、API 价格下调，同时新增专门面向 agent 工作流的 `tools` 与 `parallel_tool_calls` 字段。

## 核心观点 / 产品机制
- 上下文：1M token，与 Gemini 2.5 / DeepSeek V4 / Kimi K2.6 站到同一阵列；
- 推理：默认开启 *thinking* trace，但可通过 `reasoning_effort: low` 关闭；推理 token 单独计费，可见在 usage stats 中；
- 价格：input $1.20 / output $4.50 每百万 token，比 Grok 4.2 降 30-40%；
- 工具调用：原生支持并行 tool calls，最大 16 路；与 OpenAI 兼容 schema；
- Image / vision：保留 Grok 4 的多模态输入，输出仍为文本；
- 数据时效：截至 2026 年 4 月。

## 社区热议与争议点
- 价值派：在 1M context + agent + reasoning 三件套加持下，Grok 4.3 与 Claude Opus 4.7 / GPT-5.5 同档但便宜不少；
- 怀疑派：「xAI 的 benchmark 数字一直靠近顶端，但实际工程使用反馈两极」；
- 安全派：Grok 系列在 RLHF 上长期取向「少 refusal、多直白」，部分企业评估时认为风险高；
- 生态派：xAI 对 OpenAI / Anthropic 形成「价格 + 功能近似 + 完全独立」的第三极。

## 行业影响与未来展望
2026 年前沿 LLM 价格在 1M context + agent + reasoning 一栏的「合理价位」正在迅速下移到 $1-2 / $3-5 量级。Grok 4.3 与 DeepSeek V4 一同把这一趋势压实。对 agent 框架开发者：值得本周把 Grok 4.3 接入主流 router（OpenRouter, LiteLLM）评估，特别在长上下文 RAG、CI 调试、code review 自动化场景。对企业用户：留意 xAI 数据治理条款，特别是默认是否参与训练。

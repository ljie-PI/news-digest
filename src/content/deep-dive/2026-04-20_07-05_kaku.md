# Kaku — 为 AI 编程而生的极速终端

## 定位与痛点剖析
Kaku 是由知名开源开发者 tw93 打造的终端模拟器，基于 WezTerm 深度定制，专为 AI 辅助编程场景优化。它解决的核心痛点是：现有终端（Alacritty、Kitty、Ghostty、iTerm2 等）各有优势，但没有一个能在速度、默认配置完善度和 AI 编程工具集成之间取得完美平衡。

今日获得 +60 stars，在终端工具品类中属于稳健增长。

## 核心架构与技术细节
Kaku 是 WezTerm 的深度定制 fork，保持完整的 Lua 配置 API 兼容性，同时做了大量优化：

**性能优化**：
- 可执行文件大小从约 67MB 压缩到约 40MB（激进的符号剥离和功能裁剪）
- 资源文件从约 100MB 优化到约 80MB（资源优化 + 懒加载）
- 启动延迟从"标准"优化到"即时"（即时初始化策略）
- Shell 启动从约 200ms 优化到约 100ms

**AI 集成**：
- **错误恢复**：命令失败时自动建议修复方案，Cmd+Shift+E 一键应用
- **自然语言转命令**：在提示符处输入 `#` + 回车，LLM 将自然语言查询转换为 shell 命令
- **AI 工具配置面板**：统一管理 Claude Code、Codex、Gemini CLI、Copilot CLI、Kimi Code 等 AI 编码工具的设置
- 支持 OpenAI 兼容 API 和自定义提供商

**内置工具链**：
- Lazygit 集成（Cmd+Shift+G）
- Yazi 文件管理器（Cmd+Shift+Y）
- 内置 zsh 插件套件
- 自动深色/浅色主题切换

## 竞品对比与生态站位
| 特性 | Kaku | WezTerm | Ghostty | Warp | iTerm2 |
|------|------|---------|---------|------|--------|
| 开箱即用 | ✅ | ❌ 需配置 | ✅ | ✅ | 部分 |
| AI 集成 | ✅ 原生 | ❌ | ❌ | ✅ 原生 | ❌ |
| 二进制大小 | ~40MB | ~67MB | 小 | 大 | 大 |
| 可定制性 | ✅ Lua | ✅ Lua | 有限 | 有限 | ✅ |
| 平台 | macOS only | 全平台 | 全平台 | macOS/Linux | macOS |

Kaku 最大的限制是 **仅支持 macOS**，这限制了其受众范围，但在 Mac 生态中提供了可能是最好的 AI 编程终端体验。

## 开发者反馈与局限性
- tw93 此前的作品 Pake 和 Mole 都获得了良好口碑
- 仅支持 macOS，暂无 Windows/Linux 计划
- 作为 WezTerm fork，需要持续跟进上游更新
- AI 功能依赖外部 API，无内置模型

# Pascal Editor (+769⭐)

## 定位与痛点剖析

Pascal Editor 是一个基于 Web 的 3D 建筑编辑器，让用户在浏览器中创建和分享三维建筑项目。它瞄准的痛点是传统建筑 CAD 软件（如 AutoCAD、Revit）的高门槛、高成本和平台锁定问题。建筑师和设计师需要安装庞大的桌面软件，协作困难，而 Pascal 试图把这一切搬到浏览器里，实现零安装、实时协作的 3D 建筑设计体验。

## 核心架构与技术细节

项目采用 Turborepo monorepo 架构，分为三层：

- **@pascal-app/core**：场景数据层，使用 Zustand 管理状态，定义了完整的建筑节点层级（Site→Building→Level→Wall/Slab/Ceiling/Roof/Zone），节点以扁平字典存储（非嵌套树），通过 parentId 建立关系。支持 IndexedDB 持久化和 Zundo 实现的 50 步撤销/重做。
- **@pascal-app/viewer**：渲染层，基于 React Three Fiber + WebGPU，负责 3D 渲染、相机控制和后处理。
- **apps/editor**：Next.js 应用，UI 组件、交互工具和编辑器特有系统。

最精妙的设计是 **dirty node + System 更新模式**：节点变更时标记为 dirty，各 System（WallSystem、SlabSystem 等）在 useFrame 渲染循环中只处理 dirty 节点，避免全场景重算。墙体系统支持 CSG 布尔运算（门窗开洞）、自动 mitering（转角处理）。组件间通信使用 mitt 类型化事件总线。

Scene Registry 维护节点 ID 到 Three.js Object3D 的映射，避免场景图遍历，实现 O(1) 查找。

## 竞品对比与生态站位

直接竞品：**Speckle**（开源建筑数据平台）、**IFC.js**（开源 BIM 工具库）、**Autodesk Forma**（云端建筑设计）。Pascal 的差异化在于它是完整的编辑器而非工具库，且原生 WebGPU 支持意味着更好的渲染性能。劣势是生态尚小（769⭐），缺乏 IFC/BIM 标准格式支持的明确说明，在专业建筑行业的可用性存疑。

相比 Three.js Editor 等通用 3D 编辑器，Pascal 的建筑领域特化（墙体、楼层、区域等概念原生支持）是核心优势。

## 开发者反馈与局限性

项目发布了 npm 包（@pascal-app/core、@pascal-app/viewer），说明有意构建可复用生态。但 WebGPU 目前浏览器支持仍有限（Safari 支持不完整），可能限制实际使用。建筑行业对精度、标准格式（IFC、DWG）的要求极高，项目目前未提及这方面的支持。

# ScreenVeil

> Hide what shouldn't be seen on your computer

**Votes:** 81 | **Daily Rank:** #12 | **Topics:** Privacy, Menu Bar Apps

## 背景

随着远程办公、直播、屏幕录制和在线演示成为日常，桌面截屏中无意暴露隐私信息（聊天通知、邮件预览、密码管理器、客户名单、未发布产品）已成为高频事故。从 Zoom 通话误露 Slack 私信，到主播泄露 API key，每次"翻车"都让用户对屏幕分享更加焦虑。市面上已有 macOS 原生的"专注模式"和第三方挡板工具，但大多功能单一——只能隐藏 Dock，或只能盖住菜单栏，缺乏一个统一、可配置、可保存为预设的解决方案。ScreenVeil 在 macOS 平台切入这一痛点。

## 产品机制

ScreenVeil 是一款 macOS 菜单栏应用，核心能力是在任意 App 上方放置"浮动面板"——既可以是遮挡敏感区域的不透明蒙板，也可以是放置笔记、提示、高亮的辅助层。用户可以在几秒内创建面板，调整大小和位置，并将一整套布局保存为"预设（preset）"，下次开会、直播、演示时一键调用。典型使用场景：会议前激活"演示预设"，自动屏蔽通知中心、邮件 dock 图标，同时在演示窗口旁固定一个备注面板。它本质上是把"屏幕的视觉控制权"做成可编排、可复用的图层系统。

## 社区争议

发布日 81 票、评论较少，反馈多正面：用户称赞"presets"思路实用，比手动拉窗口快得多。质疑集中在两点——其一，是否能与系统截屏 API 做到完全兼容（即被屏蔽的内容是否在录屏中真的不可见，而不仅仅是视觉遮挡）；其二，定价模型是订阅还是买断仍是用户关心的问题。竞争上，PeekFocus（同日发布）走"模糊背景"路线，二者定位互补但用户群重叠。

## 行业影响

ScreenVeil 反映了"屏幕隐私层（Screen Privacy Layer）"作为独立赛道正在成型。过去这类需求被分散在录屏软件、会议软件、专注工具中，如今出现专门做"视觉控制"的轻量产品。对于内容创作者、技术布道师、企业 IT，这是一个低成本、强需求的新工具类别，预计未来会出现 Windows 版本和企业级合规集成。

# Cybersecurity looks like proof of work now｜深度解读

**事件背景**

这篇文章建立在一个更大的前提上：Anthropic 的安全模型 Mythos 被描述为强到足以完成复杂企业网络攻击，因此只向关键软件厂商定向开放，用于先加固系统。作者 Daniel Breunig 关注的不是“模型是否惊艳”，而是如果 AISI 的第三方评估成立，那么网络安全的经济学可能正在发生变化——安全不再主要靠人类聪明地设计，而越来越像一场算力和 token 预算竞赛。

**核心观点/产品机制**

文章的核心论点非常鲜明：**要加固系统，你可能需要花比攻击者更多的 token 去发现漏洞。**

AISI 测试的是一个名为 “The Last Ones” 的 32 步企业网络攻击模拟，估算人类需要 20 小时。测试中，Mythos、Opus 4.6 和 GPT-5.4 都拿到了 1 亿 token 预算；只有 Mythos 在 10 次尝试里成功了 3 次。更关键的是，随着 token 预算增加，模型似乎并没有明显出现收益递减。也就是说，如果继续砸钱，它可能继续找到更多漏洞。

作者由此提出一个近似 PoW 的类比：在这种世界里，防御不是“有没有足够聪明的安全工程师”，而是“你愿不愿意为系统性攻击模拟付出比攻击者更高的持续推理成本”。不靠巧思，靠的是预算、迭代次数和持续计算。

文章还给出两个推论。第一，开源软件依然重要，因为如果许多企业愿意用 token 去加固同一套 OSS，它整体上可能比你个人重写的一套功能更安全。第二，未来 agentic coding 流程会被拆成三段：开发、评审、加固。代码变便宜了，但安全代码未必便宜。

**社区热议与争议点**

HN 的讨论比原文更细。

第一类争论是：复杂系统是否一定能被“砸 token”砸出漏洞？有评论反驳说，这隐含了一个很奇怪的前提：所有代码库都是无限可挖的。类比 fuzzing，如果系统设计足够简单、边界足够刚性，比如所有网络消息都必须先验签，那再多探索也未必能突破。支持原文的人则会说，现实世界里的系统远没有那么干净。

第二类讨论是“复杂”对人类和对 LLM 是否等价。有人认为，大型代码库、不模块化、冷门语言、过于 clever 的代码，对 LLM 也一样是复杂度；另一方则举例说，LLM 已经能读 minified JS、反汇编 68000 汇编、结合 Ghidra MCP 逆向 ROM，这些都是人类很慢而模型很快的事。争议的实质是：我们到底该按人类难度，还是按模型难度来重新定义攻击面。

第三类讨论偏向工程现实：很多公司并不真的在乎安全，安全只是营销章。因为它不是主卖点，所以大家宁愿 vibe coding、赶功能、堆复杂度，也不愿为 hardening 预算买单。这一点其实反过来强化了作者的判断：即便“加固阶段”成为新范式，愿意长期为此付费的组织可能仍是少数。

**行业影响与未来展望**

如果作者判断成立，安全行业会出现一个非常明显的结构变化：从以人工审计、渗透测试为主的离散服务，变成以持续代理化加固、预算化攻击模拟为主的常态流程。对软件团队而言，安全可能像 CI 一样变成一个持续烧钱但不能省的步骤。

但这也会造成更强的马太效应：大厂和关键基础设施运营者更有能力持续投入 token，普通中小团队可能只能做到“够卖就行”的最低安全水平。届时，安全差距会更像资本差距，而不是单纯的工程水平差距。

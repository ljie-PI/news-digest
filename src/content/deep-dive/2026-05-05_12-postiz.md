# Postiz — Agentic social media scheduler for agents

## 1. 事件背景
社媒发布工具市场非常拥挤（Buffer、Hootsuite、Later、Typefully…），但传统产品都是「人在 GUI 里点」的工作流。在 Agent 时代，越来越多用户希望把社媒发布让 Claude / OpenClaw / Cursor 等 AI 直接接管，而不是手工排期。Postiz 在 Product Hunt 拿到 488 票（weekly），定位非常具体：**为 Agent 而生的社媒调度器**，号称支持 30+ 社交网络、能一次排上千条贴文，并显式集成主流 AI Agent 平台。

## 2. 产品机制
Postiz 提供两条接入路径：**Postiz SKILL + Postiz CLI**，可被 OpenClaw、Claude、nanoclaw、PaperClip 等 Agent 直接调用；或 **Postiz MCP（Model Context Protocol）服务器**，连接到任何远程 Agent runtime。底层覆盖 X/Twitter、LinkedIn、Threads、Mastodon、Bluesky、TikTok、Instagram、Pinterest、YouTube、Discord、Reddit 等 30+ 渠道，统一抽象账号、媒体上传、定时发布、跨平台变体（针对每个渠道字符限制 / 媒体规格自动调整）。开源版 + 托管 SaaS 的双轨模式。

## 3. 社区热议与争议点
正方 Pros：
- Maker 圈高度认可「Agent-first 而非 GUI-first」的产品哲学，认为这是多平台社媒工具的下一阶段。
- 30+ 渠道 + MCP 接入对自动化营销和 KOC 矩阵管理非常实用。

反方 Cons：
- 部分 PH 评论提到 Instagram、TikTok 的官方 API 历来反复变动，Postiz 是否能稳定支撑发文 + 媒体上传值得观察。
- 与 Buffer、Typefully 这类老牌玩家相比，企业合规、团队权限、审批流仍待补齐。
- Agentic 自动发文同时引出「AI 内容垃圾化」担忧，社区讨论希望 Postiz 内置内容质量与防 spam 检测。

## 4. 行业影响与未来展望
Postiz 的窜红代表了 **「MCP 化 + Agent-first 重做 SaaS」** 的清晰潮流：传统 SaaS 的 GUI 不再是必经路径，Agent 通过 MCP/CLI 调用即可完成业务动作。社媒发布、CRM、邮件营销、广告投放都会陆续被以同样方式重构。短期 Postiz 会争夺 Indie Hacker 与 KOC 群体；长远，谁能把发布质量、品牌一致性、合规审计纳入 Agent 路径，谁就能拿到中大型客户。

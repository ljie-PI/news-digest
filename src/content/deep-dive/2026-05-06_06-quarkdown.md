# iamgio/quarkdown — 把 Markdown 变成图灵完备排版语言

- 仓库：https://github.com/iamgio/quarkdown
- 时间段新增：本周 +2,557 stars（总 13,667，主语言 Kotlin）

## 1. 定位与痛点剖析

LaTeX 排版强但语法陡峭，Markdown 写作顺手却几乎无法做严肃排版（多栏论文、PPT、印刷书）。Quarkdown 直接把 Markdown 升级为一门"图灵完备的排版扩展语言"：从一份源码即可编译为可印刷书籍、学术论文、知识库站点或交互式 PPT。目标用户是研究者、技术写作者、独立出版人——他们想要 Markdown 的写作体验 + LaTeX/Typst 级别的产出质量。

## 2. 核心架构与技术细节

主语言 Kotlin，自定义解析器在 CommonMark + GFM 之上扩展了"函数调用"语法：`.somefunction {arg1} {arg2}` 可作为内联指令调用任意 Quarkdown 函数，并支持 body 参数。这一设计相当于把 Pandoc 的"过滤器"思路从外挂提到一等公民，加上变量、控制流、自定义模板，于是诞生了图灵完备能力。编译目标包括 PDF（基于 LaTeX 后端）、HTML、reveal.js 风格 PPT 等。CLI 通过 `quarkdown` 命令完成 init/compile，配套有 mock 文档与脚手架，便于快速生成项目骨架。

## 3. 竞品对比与生态站位

直接对标 Typst（Rust 实现、新型排版语言）、Pandoc + Lua filter、MyST/Sphinx、Quarto。相对 Typst：Quarkdown 完全 Markdown-flavored，更亲近现有 .md 资产；相对 Pandoc：把扩展能力做成一等公民，不再依赖外部脚本；相对 Quarto：路线更通用，不绑定 R/Python notebook。劣势是 Kotlin/JVM 启动慢、生态小、字体/排版细节远不如 Typst 与 LaTeX 成熟。

## 4. 开发者反馈与局限性

近期 issue 高频话题：中文/CJK 字体支持、PDF 后端选择（LaTeX 依赖体积大）、PPT 模板可定制度。开发者偏爱的点是"一份 .qmd 同时出书 + PPT + 站点"的产出灵活性，并对函数式扩展赞誉颇多；批评点是文档主要面向英语场景，复杂数学公式与参考文献管理仍需更多打磨。

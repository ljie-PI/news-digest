# ClawSecure — 给 AI Agent 的"杀毒软件" — 深度解读

## 产品定位

**ClawSecure** 把自己包装成"AI Agent 的杀毒 + EDR",PH 当周 293 票、daily rank #5,主打一个让人印象深刻的数据:**41% of top agents are dangerous.** 它不是又一个 prompt-injection 检测库,而是把"传统终端安全"那套——预安装扫描、运行时监控、伴生 Security Companion、Verification API——整体平移到 Agent 这一新计算单元上。

## 三层防御架构

1. **Pre-install Scan**:在 Agent/技能/MCP server 安装前静态分析其代码、prompt、tool schema,检测已知 prompt-injection、数据外泄模式、隐蔽工具调用——号称对 OWASP **ASI(Agent Security Initiative)** Top 10 全覆盖。
2. **Runtime Monitor**:挂在 Agent 进程或 sidecar,实时拦截可疑工具调用、敏感数据流出、上下文污染;支持回放与时间线 diff。
3. **In-Agent Companion**:一个轻量的 LLM agent 与主 Agent 共驻,以"安全顾问"角色在每次重大动作前介入审计;
4. **Verification API**(亚 200 ms):给第三方 Agent 平台/MCP host 提供一行 RPC 校验接口,可挂在每次 tool 调用前。

## 时机判断

ClawSecure 出现的时点恰好契合三个趋势:
- **MCP 大爆发**:2025-2026 年 MCP server 数量激增,鱼龙混杂,社区已多次曝出恶意 MCP 窃取 token 的案例;
- **Agent 自主度上升**:从"copilot"走向"autonomous worker",误操作的 blast radius 急剧放大;
- **OWASP ASI 标准化**:OWASP 在 2025 年底正式发布 Agent Security Top 10,给安全产品提供了清晰的对标坐标。

## 商业模式与竞争

免费 + 无需注册的 freemium 是典型 PLG 打法,目标是先占领开发者心智,再向企业卖 Verification API/SOC dashboard。同赛道竞品包括 Lakera、Protect AI、Robust Intelligence 的 Agent SKU,以及 HiddenLayer 的 ModelScanner;ClawSecure 的差异化在于"全栈式 Agent 视角"而非纯 model/prompt 视角,以及与新兴 OpenClaw 生态的深度绑定。

## 风险与质疑

PH 评论区已有质疑:"41% 危险"如何定义?是否存在选择性样本制造焦虑?另外 In-Agent Companion 本身也是一个 LLM,如何避免它被同样手法 prompt-inject?长远看,Agent 安全大概率被超大平台(Anthropic/OpenAI/Google)以默认能力收编,独立厂商需要尽快锁定 enterprise compliance 通道(SOC2 audit、合规报告自动化)以建立壁垒。对国内做 Agent 平台的团队,这是一个值得马上预研的方向。

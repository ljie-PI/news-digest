# sem — 语义版本控制 CLI

## 定位与痛点剖析
sem 是构建在 Git 之上的语义版本控制工具。它解决了一个开发者长期忍受的痛点：git diff 只告诉你"第 X-Y 行变了"，而 sem 告诉你"函数 blah 被修改了"。传统的行级别 diff 在代码审查时缺乏语义信息——你不知道一个改动影响了哪些函数、类或方法，也无法直观看到重命名和移动操作。

## 核心架构与技术细节
- **技术栈**：Rust + tree-sitter（支持 24 种编程语言的 AST 解析）
- **三阶段实体匹配**：(1) 精确 ID 匹配 → 修改/未变 (2) 结构哈希匹配 → 重命名/移动（忽略空白和注释） (3) 模糊相似度 → >80% token 重叠视为可能的重命名
- **核心命令**：
  - `sem diff`：实体级 diff，支持 JSON/Markdown/Plain 输出
  - `sem impact`：跨文件依赖图，显示改动影响范围
  - `sem blame`：实体级 blame，显示谁最后修改了每个函数
  - `sem log`：追踪单个实体的 git 历史演变
  - `sem context`：为 LLM 提供 token 预算内的上下文
  - `sem setup`：替换 git diff 输出，无需改变任何命令即可获得语义 diff
- **MCP 服务器**：6 个工具供 AI Agent 使用
- **输出格式**：支持 JSON（CI/AI Agent）、Markdown（PR 报告）、Plain（终端）

## 竞品对比
- **vs git diff**：从行级别升级到实体级别，支持重命名检测
- **vs GitHub/GitLab 的语义 diff**：sem 是本地工具，不依赖平台，且提供 impact analysis
- **vs Difftastic**：Difftastic 做结构化 diff 但不做实体提取和影响分析

## 开发者反馈与局限性

# imputnet/helium — 注重隐私的 Chromium 派生浏览器

- 仓库：https://github.com/imputnet/helium
- 时间段新增：本周 +773 stars（总 15,287）

## 1. 定位与痛点剖析

Chrome 默认追踪、Edge 强推 AI 弹窗、Brave 的 BAT 与广告替换被部分用户视作"换汤不换药"，社区一直在找一款"无 bloat、原生隐私、跨平台、和 Chromium 上游接近"的浏览器。Helium 把目标定为 "made for people, with love"：精简 + 内建无偏广告拦截 + 无遥测，覆盖 macOS / Linux / Windows。目标用户是注重隐私但不愿放弃 Chromium 兼容性的高级用户、开发者、记者等。

## 2. 核心架构与技术细节

基于 Chromium 修改的多平台浏览器。仓库以 monorepo 风格管理：核心 Helium、平台打包仓库（Helium for macOS / Linux / Windows）、Web 服务（Helium services）、引导页（Helium onboarding）等独立子仓。所有打包、工具、服务均开源在 GitHub 上，用户可自行审计构建。当前处于 Beta，下载从官网 helium.computer 自动选择平台二进制，亦可在 GitHub Releases 抓取源码版本。

## 3. 竞品对比与生态站位

直接竞品：Brave、Vivaldi、Arc、LibreWolf。相对 Brave：Helium 不引入加密货币奖励、广告拦截无偏（不替换为自家广告网络）；相对 LibreWolf：Helium 走 Chromium 路线，能原生兼容 Web Store 扩展和 Chrome DevTools；相对 Arc：Arc 已停更转 Dia，Helium 占住了"Arc 难民"的位置之一。

## 4. 开发者反馈与局限性

社区议论：性能与内存占用（Chromium 改进型常见短板）、与 Manifest V3 的妥协、企业策略 GPO/MDM 支持、扩展商店替代方案。已知局限：仍处 Beta，部分平台二进制 / 自动更新链路未稳定；隐私默认值与 Chromium 同步策略之间的取舍仍在演进。

# ghostty-org/ghostty — 用 Zig 写的"快+原生+功能丰富"终端

- 项目链接: https://github.com/ghostty-org/ghostty
- 官网: https://ghostty.org
- 时间段新增 stars: +1,731 (this week)

## 1. 定位与痛点剖析

Ghostty 是 Mitchell Hashimoto（HashiCorp 创始人）个人主导的开源终端模拟器。痛点定位非常清晰——市场上的终端各有各的问题：iTerm2 慢、Alacritty 没原生 UI、kitty 不是真原生应用、Terminal.app 落后。Ghostty 试图同时拿到"快、功能丰富、原生 UI"三个维度——而绝大多数终端只能拿到其中两个。它瞄准的是终端重度用户：Vim/tmux 党、SRE、系统编程人员。

## 2. 核心架构与技术细节

主体由 Zig 编写，核心可复用为 C/Zig 库 **libghostty**——任何人都能用它构建自己的终端，或把终端能力嵌入到自己应用里（参考实现 Ghostling）。架构亮点：
- **多线程**：每个终端独立 read/write/render 三线程；
- **渲染器**：Linux 用 OpenGL，macOS 用 Metal + CoreText 字体发现；
- **Parser**：CPU SIMD 优化的状态机；
- **协议覆盖**：实现完整的 xterm legacy 序列 + 现代协议（Kitty graphics protocol、Kitty image protocol、clipboard sequences、synchronized rendering、light/dark mode notifications）；
- **平台原生**：macOS 是真 SwiftUI 应用，含菜单栏 / 设置 GUI / AppleScript / Apple Shortcuts（AppIntents）；Linux 是真 GTK 应用。

性能上号称与 Alacritty 在 benchmark 上几个百分点之差，但比 Terminal.app/iTerm 快约 100 倍。

## 3. 竞品对比与生态站位

紧邻对手 **Alacritty**（Rust，无原生 UI）、**Kitty**（自渲染，多平台但非原生）、**Wezterm**（Rust，Lua 配置）、**Warp**（同周 trending 第 1，agent 化路线）。Ghostty 与 Warp 形成有趣对照——同期登榜，但路线相反：Warp 把终端"AI 化"，Ghostty 故意只做"更纯粹更快"的终端。Hashimoto 在多次访谈中强调 Ghostty **不会内置 AI agent**，把"AI 协作"外推给用户自选 CLI。

## 4. 开发者反馈与局限性

社区评价非常正面：性能、原生体验、Roadmap 透明（README 内置 6 步 roadmap，前 5 步打勾，第 6 步"Ghostty-only 控制序列"未完成）。
局限：
- **Windows 缺席**——只有 macOS/Linux，长期是用户最大抱怨点；
- **配置面**：相比 Wezterm 的 Lua 全局可编程，Ghostty 配置语法较保守；
- **Zig 仍 0.x**：升级 Zig 工具链时常打破构建；
- **某些 GTK 主题适配差**——Linux 美观度不如 macOS。

总体来看，Ghostty 在"日常生产力终端"赛道是当下最值得装的选项之一，特别是 macOS 用户。

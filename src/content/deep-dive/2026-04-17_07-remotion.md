# Remotion — 用 React 编程式创建视频

> ⭐ 日增 +192 stars | 作者: remotion-dev

## 1. 定位与痛点剖析

Remotion 是一个 **基于 React 的编程式视频制作框架**，让开发者用写 React 组件的方式来创建视频。它解决的痛点：

- **视频制作门槛高**：传统视频编辑需要 After Effects、Premiere 等专业软件，学习曲线陡峭
- **重复性视频无法自动化**：产品宣传、数据报告、社交媒体内容等需要批量生产
- **设计师与开发者协作困难**：视频制作通常是设计师的领域，开发者难以参与
- **动态内容视频化**：API 数据、实时数据需要可视化为视频格式

Remotion 的理念是：如果你会写 React，你就会做视频。

## 2. 核心架构与技术细节

**技术栈**：React + TypeScript，Node.js 生态。

核心设计思路：**每一帧视频就是一个 React 组件的渲染结果**。框架负责：
- 按帧率调用组件渲染
- 将渲染结果捕获为图片
- 编码为视频文件

开发者可以使用全部 Web 技术：CSS 动画、Canvas、SVG、WebGL、任何 npm 包。

```bash
npx create-video@latest  # 快速开始
```

关键特性：
- **React 生态复用**：组件复用、状态管理、强大的组合能力
- **Fast Refresh**：开发时实时预览，修改代码即时看到效果
- **编程式控制**：变量、函数、API 调用、数学算法驱动动画
- **多种渲染方式**：本地 CLI、Node.js API、AWS Lambda、Cloud Run
- **丰富的内置功能**：字幕、3D（Three.js）、图表、文字效果、转场、媒体处理

Remotion 支持复杂的渲染管线——可以在 Lambda 上并行渲染数千个视频，适合大规模内容生产。

## 3. 竞品对比与生态站位

**vs FFmpeg**：
- FFmpeg 是底层工具，功能强但无法处理复杂动画和布局
- Remotion 站在更高抽象层，用 React 组件描述视频内容

**vs After Effects / Motion**：
- 专业视频工具功能更全面
- Remotion 的优势在于自动化、版本控制、CI/CD 集成

**vs Lottie / Motion Canvas**：
- Lottie 主要用于动画导出，不是完整视频框架
- Motion Canvas 是竞品，但生态和社区规模小于 Remotion

**vs Editframe / Shotstack**：
- SaaS 视频 API，Remotion 是自托管开源方案
- Remotion 更灵活，但需要自己管理渲染基础设施

Remotion 在编程式视频制作领域已经是事实上的标准，npm 周下载量稳定在高位。

## 4. 开发者反馈与局限性

社区非常活跃，Discord 参与度高。许多公司用于自动化视频生产。

局限性：
- **特殊许可证**：非标准开源许可，企业使用可能需要购买商业许可
- **渲染性能**：复杂视频渲染耗时较长，需要优化或使用云端并行
- **学习曲线**：需要同时理解 React 和视频制作概念
- **不适合手动剪辑**：纯编程驱动，缺乏传统 NLE 的手动编辑体验
- **浏览器引擎限制**：某些视频效果受限于 Chrome 渲染能力

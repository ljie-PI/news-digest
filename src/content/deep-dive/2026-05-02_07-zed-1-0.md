# Zed 1.0

> High-performance, open source, multiplayer code editor — 245 votes

## 1. 事件背景
Zed 终于在 2026 年 5 月以 "1.0" 正式版登场，距离它在 2022 年首次公开预览已有近 4 年。这个时间点意义重大：VS Code 长期占据编辑器市场后，Cursor、Windsurf 等 AI fork 把"AI 原生体验"卷到了极致，但底层仍是 Electron。Zed 反其道而行——用 Rust 从零写一个 GPU 加速、原生多人协作、支持并行 Agent 的编辑器，目标是把 AI Coding 时代的"承载层"重写一次。1.0 的发布让 Zed 从"小众极客玩具"正式进入主流候选名单，也是开源 IDE 阵营对 Microsoft 的一次正面冲击。

## 2. 核心观点 / 产品机制
Zed 1.0 的三大支柱：**Fast** —— 全 Rust 实现，UI 层用自研 GPUI 框架直接调用 GPU 渲染；多核架构让百万行代码搜索、跳转毫秒级响应。**Agentic** —— 内置并行 Agent 系统，可以同时跑多个 AI 任务（编辑文件、调用工具、运行测试），与 Claude Code/Codex 等 CLI Agent 通过 ACP 协议互通。**Collaborative** —— 内置 channels、屏幕共享、实时协作编辑，无需 Live Share 这类插件。商业模式上，Zed 走开源 + AI Pro 订阅路线（Claude/GPT 调用走自家网关），1.0 同步发布稳定 API、扩展商店和企业自托管版本。

## 3. 社区热议与争议点
6 条评论但 38 条 reviews（4.95/5），表明深度用户极多：(a) Rust 社区欢呼"终于有了一个不卡的 IDE"；(b) 多名前 Atom 用户（Zed 创始团队来自 Atom）情绪化点赞"GitHub 没做完的事 Zed 做完了"；(c) 争议来自 VS Code 阵营——批评 Zed 扩展生态薄弱，TypeScript/Java 等大型语言体验仍弱于 VS Code；(d) AI 派则讨论 Zed 的 Agent 是否能匹敌 Cursor 的 Tab 体验，目前评测显示 Zed 在多 Agent 并行上更强、单点补全略弱。

## 4. 行业影响与未来展望
Zed 1.0 是开源编辑器市场的"分水岭事件"。它证明了原生 IDE 体验不仅可行，而且在 AI Agent 时代有结构优势——Electron 的内存与渲染瓶颈在并行 Agent 场景下被放大。预计未来 12 个月，更多团队会评估 Zed 替代 VS Code，尤其是 Rust/Go/Web 团队。中长期，Zed 有望与 Cursor 形成"开源 vs 闭源"双雄格局；而 VS Code 阵营或被迫加速原生重写。最大风险仍是扩展生态：能否吸引到关键 LSP/插件作者，将决定它能否突破"小众精品"上限。

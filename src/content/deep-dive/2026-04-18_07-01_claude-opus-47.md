# Claude Opus 4.7 — Anthropic 最新旗舰编程模型

## 事件背景

2026年4月16日，Anthropic 正式发布 Claude Opus 4.7，这是继 Opus 4.6 之后的又一次重大迭代。当前 AI 编程助手赛道竞争激烈——OpenAI 的 Codex/GPT-5.4、Google 的 Gemini 3.1 Pro 都在争夺开发者市场。Anthropic 选择在此时推出 Opus 4.7，核心定位是"高难度软件工程任务的可信赖代理"，瞄准的是之前需要人类密切监督的复杂、长周期编码工作。

值得注意的是，Anthropic 上周刚公布了 Project Glasswing（关于 AI 网络安全风险的研究），并表示其最强模型 Claude Mythos Preview 仍限制发布。Opus 4.7 是首个搭载新型网络安全防护机制的模型——自动检测并阻止高风险网络安全请求，为 Mythos 级模型的最终广泛发布铺路。

## 核心观点/产品机制

**基准测试全面提升：**
- SWE-bench Verified：87.6%（Opus 4.6 为 80.8%）
- SWE-bench Pro：64.3%（Opus 4.6 为 53.4%）
- Cursor 编程基准：70%（Opus 4.6 为 58%）
- Rakuten-SWE-Bench：解决的生产任务量是 Opus 4.6 的 3 倍
- BigLaw Bench（Harvey 法律推理）：高 effort 下达 90.9%
- 超越 GPT-5.4 和 Gemini 3.1 Pro

**关键新特性：**
- **xhigh effort 级别**：介于 high 和 max 之间的新推理强度，Claude Code 已将其设为默认
- **/ultrareview 命令**：Claude Code 中的并行多代理 PR 审查功能
- **视觉能力大幅升级**：长边分辨率提升至 2,576px（约 3.75MP），约为之前的 3 倍
- **新分词器**：相同文本映射到最多 1.35 倍的 token，速率限制已相应上调
- **文件系统记忆**：跨多会话工作时可保留笔记，减少每次新会话的上下文粘贴
- **任务预算（公测）**：限制长时间运行的 token 消耗

定价保持不变：$5/百万输入 token，$25/百万输出 token，标准费率下支持完整的 1M token 上下文窗口。可通过 Claude API、Amazon Bedrock、Google Vertex AI、Microsoft Foundry 使用。

## 社区热议与争议点

**1. 指令遵循更严格，引发调优阵痛**
Opus 4.7 比 4.6 更加字面化地遵循指令。社区中最常见的抱怨是：之前为 Opus 4.6 调校的 system prompt 和 CLAUDE.md 配置文件需要重新调整。部分开发者反馈升级后行为"过于严格"，失去了一些灵活性。

**2. 编程能力 vs 通用能力的取舍**
Anthropic 明确指出 Opus 4.7 "不如 Mythos Preview 那样广泛能力突出"，但在编程和代理推理方面更强。这引发了关于"专精 vs 通用"路线的讨论。部分用户认为这是务实选择，另一部分人担心模型正在变成"只会写代码的工具"。

**3. 网络安全限制的双刃剑**
新的网络安全防护自动阻止可疑请求，虽然安全研究人员可以申请 Cyber Verification Program，但社区对误判率和申请门槛存在疑虑。渗透测试人员需要额外的验证步骤，增加了合规负担。

**4. 金融科技领域的早期好评**
来自 Intuit 等金融科技公司的早期测试反馈积极——模型在规划阶段能自行发现逻辑错误，加速执行过程。这对对准确性要求极高的金融领域来说意义重大。

## 行业影响与未来展望

Opus 4.7 的发布巩固了 Anthropic 在 AI 编程助手领域的领先地位。87.6% 的 SWE-bench 分数创下新高，意味着自主解决真实开源仓库问题的能力已接近实用门槛。这对整个软件工程行业的影响深远：初级代码审查、bug 修复、测试编写等工作正加速自动化。

更重要的是，Opus 4.7 的网络安全防护机制预示着 AI 安全治理的新范式——在模型层面内置安全过滤，而非依赖外部审查。如果这套机制在实际部署中验证成功，将为 Mythos 级超强模型的发布铺平道路。

竞争格局方面，OpenAI 同日更新了 Codex，Google Gemini 3.1 Pro 也在快速追赶。2026 年下半年，AI 编程工具的竞争将进入白热化阶段。
